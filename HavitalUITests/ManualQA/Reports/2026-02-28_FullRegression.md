## 問題清單

### BUG-01 [Major] 登出後未導向 LoginView

**嚴重度**: Major

**重現步驟**:
1. 在個人資料頁點擊「登出」
2. 觀察畫面

**預期行為**: 登出後清除 session，自動跳回 LoginView

**實際行為**: 顯示 token expired 錯誤，頁面停留在主畫面，需要手動重啟 App 才回到 LoginView

**建議修復**: 在 logout handler 中確保清除所有 session state 並導向 LoginView

---

### BUG-02 [Major] Authentication token 過期無自動更新

**嚴重度**: Major

**重現步驟**:
1. 使用一般帳號（非 Demo）登入
2. 閒置一段時間後操作 API
3. 觀察錯誤提示

**預期行為**: Token 過期時自動 refresh，或彈出重新登入提示

**實際行為**: 顯示 "Authentication token has expired" 錯誤，用戶卡住無法使用

**建議修復**: 實作 token refresh 機制（interceptor），過期時自動更新或導回登入

---

### BUG-04 [Minor] 心率區間邊界值顯示不一致

**嚴重度**: Minor

**重現步驟**:
1. 進入個人資料 → 查看心率區間摘要（顯示 183 bpm）
2. 進入心率區間詳情 → 查看 Zone 5 上限（顯示 184 bpm）

**預期行為**: 同一個最大心率值在兩個頁面顯示一致

**實際行為**: 個人資料頁顯示 183，詳情頁顯示 184（差 1 bpm）

**建議修復**: 確認兩個頁面使用同一個數據來源，統一四捨五入邏輯

---

### BUG-05 [Minor] 產生下週課表後跳到錯誤的週數

**嚴重度**: Minor

**重現步驟**:
1. 在訓練主頁切換到「下週」
2. 點擊「產生下週課表」
3. 觀察產生後的 view 停在哪週

**預期行為**: 產生後停在剛產生的第 2 週課表

**實際行為**: View 跳至第 3 週而非第 2 週

**建議修復**: 課表產生完成後，明確跳回剛產生的週數
