# 測試計劃：訓練方案組合矩陣 UI 驗證

## 目標

透過 iOS Simulator MCP 操作 UI，驗證各目標類型 × 方法論組合的 onboarding 流程與課表內容合理性。

---

## 測試矩陣

| # | 目標類型 | 方法論 | 距離/週數 | 預期特徵 |
|---|--------|--------|---------|---------|
| T1 | 賽事訓練 (race_run) | Paceriz 平衡訓練法 | 全馬 42K / 4h | 基礎→增強→巔峰→減量，均衡配速訓練 |
| T2 | 賽事訓練 (race_run) | 挪威乳酸閾值訓練法 | 半馬 21K / 1h50m | 4x4min 間歇為主，乳酸閾值重點 |
| T3 | 賽事訓練 (race_run) | 漢森馬拉松訓練法 | 全馬 42K / 4h | 累積疲勞法，較短長跑但更高頻 |
| T4 | 賽事訓練 (race_run) | 極化訓練法 (80/20) | 10K / 50min | 80% 低強度 + 20% 高強度 |
| T5 | 新手入門 (beginner) | (API 預設) | 預設週數 | 漸進式增量，無高強度 |
| T6 | 維持訓練 (maintenance) | (API 預設) | 12 週 | 穩定跑量，維持體能 |

---

## 統一測試參數

- **個人最佳 (PB)**：關閉（跳過）
- **週跑量**：使用預設值
- **訓練天數**：週二/四/六（3 天）
- **長跑日**：週六

---

## 每個測試案例的驗證項目

### A. Onboarding 流程

| 項目 | 說明 |
|------|------|
| A1 | 目標選擇頁面正確顯示 |
| A2 | 方法論選項與目標類型匹配 |
| A3 | 訓練天數/長跑日選擇正常 |
| A4 | 起始階段推薦合理（根據週數） |

### B. 訓練總覽頁

| 項目 | 說明 |
|------|------|
| B1 | 總週數合理（賽事：依距離 8-20 週；非賽事：用戶設定） |
| B2 | 階段時間軸正確（順序、週數分配） |
| B3 | 目標配速顯示正確 |
| B4 | 目標評估文字合理 |
| B5 | 訓練方針描述與所選方法論一致 |

### C. 週課表驗證（第一週）

| 項目 | 說明 |
|------|------|
| C1 | 每日訓練類型合理（有休息日、有長跑日） |
| C2 | 訓練量與階段匹配（基礎期不應有超高強度） |
| C3 | 配速建議在合理範圍（3:00-8:00/km） |
| C4 | 暖身/主訓/收操結構完整 |
| C5 | 間歇訓練有正確的組數/時間/恢復 |
| C6 | 週總跑量合理 |

### D. 合理性

| 項目 | 說明 |
|------|------|
| D1 | 課表對目標跑者「看起來合理」 |
| D2 | 不會出現離譜配速（<2:30/km 或 >9:00/km） |
| D3 | 休息日安排合理（不連續高強度） |
| D4 | 長跑日在用戶選的那天 |

## 測試執行步驟

```
1. 從個人資料 → ⋯ 選單 → 訓練總覽 → 重新設定目標（或首次 onboarding）
2. 走 onboarding 流程：
   a. 個人最佳成績 → 關閉 toggle（跳過）
   b. 週跑量 → 使用預設
   c. 選擇目標類型
   d. 設定訓練目標（距離+時間 或 週數）
   e. 選擇起始階段（若出現）
   f. 選擇方法論（若出現）
   g. 選擇訓練日（週二/四/六，長跑日=週六）
   h. 確認訓練總覽 → 產生課表
3. 截圖訓練總覽
4. 進入第一週課表 → 逐日展開查看
5. 截圖每日訓練內容
6. 產出測試報告到 Reports/
```

## 測試優先級

1. **T1** — 最常見場景（全馬 + Paceriz），作為基準
2. **T2** — 驗證挪威法差異（間歇結構）
3. **T5** — 新手入門（完全不同的流程路徑）
4. **T3** — 漢森法差異
5. **T4** — 極化訓練差異
6. **T6** — 維持訓練
