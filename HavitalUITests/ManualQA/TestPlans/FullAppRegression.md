# 測試計劃：全 App Regression 測試

## 目標

透過 iOS Simulator MCP 操作 UI，全面驗證 Paceriz app 所有主要功能的 UI 正確性、流程完整性與資料合理性。

---

## 前置條件

- Simulator 已啟動，app 已安裝並可執行
- 測試帳號已備妥（email/password）
- 每個測試模組可獨立執行，但建議依 S → R 順序

## S — 登入 / 登出

### 測試步驟

```
1. 冷啟動 app
2. 確認 LoginView 顯示
3. 點擊「以 Email 登入」
4. 輸入測試帳號 email + password
5. 點擊登入
6. 確認進入主畫面（TrainingPlanView）
7. 進入個人資料頁 → 點選登出
8. 確認返回 LoginView
```

### 驗證項目

| 項目 | 說明 |
|------|------|
| S1 | LoginView 正確顯示 app 名稱「Paceriz」 |
| S2 | Email 登入表單可輸入 |
| S3 | 登入成功後不再顯示 LoginView |
| S4 | 登入成功後顯示用戶名稱或個人化資料 |
| S5 | 登出後清除狀態，返回 LoginView |
| S6 | 錯誤密碼應顯示錯誤提示（非 crash） |

## O — Onboarding 完整流程

> 參考：TrainingPlanMatrix.md，執行 T1（全馬 + Paceriz 平衡訓練法）作為基準 onboarding 驗收

### 測試步驟

```
1. 個人資料 → ⋯ 選單 → 訓練總覽 → 重新設定目標
   （或首次登入觸發）
2. 走完以下步驟：
   a. 個人最佳頁 → 關閉 toggle（跳過 PB）
   b. 目標類型選擇 → 「賽事訓練」
   c. 距離選擇 → 「全馬 42K」
   d. 目標時間 → 4:00:00
   e. 起始階段 → 接受推薦
   f. 方法論 → 「Paceriz 平衡訓練法」
   g. 週跑量 → 使用預設
   h. 訓練天數 → 週二、週四、週六（3 天）
   i. 長跑日 → 週六
   j. 確認訓練總覽 → 產生課表
```

### 驗證項目

| 項目 | 說明 |
|------|------|
| O1 | 每個步驟有「返回」可回上一步 |
| O2 | 距離選項包含：5K / 10K / 半馬 / 全馬 / 超馬 |
| O3 | 時間輸入格式正確（HH:MM:SS 或 MM:SS） |
| O4 | 方法論選項顯示合理描述文字 |
| O5 | 訓練天數可多選，長跑日只能選已選的天 |
| O6 | 訓練總覽正確反映所有選擇 |
| O7 | 「產生課表」按鈕有 loading 狀態 |
| O8 | 完成後進入訓練主頁，顯示第一週課表 |

## T — 訓練計劃主頁

### 測試步驟

```
1. 確認在主訓練頁（TrainingPlanView）
2. 查看當週課表總覽
3. 切換到下一週
4. 切換回當週
5. 展開今日訓練卡片
6. 確認「週進度」顯示正確
```

### 驗證項目

| 項目 | 說明 |
|------|------|
| T1 | 當週日期範圍顯示正確 |
| T2 | 七天的訓練 / 休息 / 完成狀態正確 |
| T3 | 今日有高亮或特殊標記 |
| T4 | 週切換（← →）動作流暢，日期更新 |
| T5 | 週進度 bar 百分比合理（0–100%） |
| T6 | 今日訓練卡片（TodayFocusCard）顯示訓練名稱與配速 |
| T7 | 休息日不顯示配速資訊 |
| T8 | 訓練日期不會顯示成 1970 或亂碼 |

## 測試執行優先順序

```
P0（每次 release 必測）:
  S → O（T1 全馬基準）→ T → D

P1（每週回歸）:
  W → P → H → V

P2（雙週回歸）:
  N → M → C

P3（月度或重大更新後）:
  A → R
```

---

## 如何呼叫 ui-qa 執行此計劃

```
按照 HavitalUITests/ManualQA/TestPlans/FullAppRegression.md 執行 P0 模組
（或指定：執行模組 S、O、T、D）
```
